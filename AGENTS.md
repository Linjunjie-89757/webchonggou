# AGENTS.md

本文件定义本项目中 AI 编码代理的协作规则。目标是保护现有代码和中文文案，保持最小改动，并在修改后可验证、可回退。

## 基本原则

- 先理解当前需求、代码结构和已有实现，再动手修改。
- 只修改与当前目标直接相关的文件和代码，不顺手重构无关内容。
- 优先沿用项目现有技术栈、目录结构、命名方式、组件风格和接口习惯。
- 遇到用户已有未提交改动时，不得擅自覆盖、回退或删除；必须在当前内容基础上做增量修改。
- 修改后尽量运行与改动匹配的验证命令；无法验证时，需要说明原因和风险。

## 中文文件编码保护（强制）

本项目包含中文文案、中文注释、中文配置项和中文目录。修改这类文件时，必须保护原始文件编码。

- 不得使用 PowerShell 的 `Get-Content` / `Set-Content` / 管道 / 重定向对含中文文件进行整文件读写替换。
- 不得在未确认编码的情况下批量重写 `.vue`、`.ts`、`.tsx`、`.js`、`.json`、`.md`、`.yml`、`.properties` 等文件。
- 小范围改动必须优先使用 `apply_patch`，只修改必要行，避免改变文件原始编码、BOM 和换行风格。
- 批量改动必须先识别并保持原编码；如果无法可靠识别，先询问用户。
- 修改前检查 `git status`；存在大量未提交改动时，先提醒用户备份或提交。
- 修改后检查 `git diff --check` 和关键中文文本，确认没有中文乱码、BOM 丢失、换行风格异常或无关整文件格式化。

## 禁止事项

- 禁止使用 `git reset --hard`、`git checkout --`、`git restore` 等可能丢失用户改动的命令，除非用户明确要求。
- 禁止为了批量替换而整文件重写中文源码或配置文件。
- 禁止在没有确认成功的情况下声称已经完成。
- 禁止把无关格式化、依赖锁文件变化、构建产物变化混入当前任务。
