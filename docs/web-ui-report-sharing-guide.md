# Web UI 报告分享使用说明

## 适用范围

本文说明 Web UI 自动化报告的两种分享方式：

- 非公开链接：需要登录平台后访问。
- 公开分享链接：不需要登录，任何拿到链接的人都可以只读查看报告。

公开分享适合发给同事、CI 构建结果、临时排查人员。涉及敏感业务截图时，优先使用非公开链接。

## 非公开链接

在运行报告或批次报告中点击“复制非公开链接”。

非公开链接特点：

- 必须登录平台。
- 继续使用当前账号和工作空间权限。
- 可以打开指定运行报告或批次报告。
- 适合团队内部日常协作。

示例：

```text
/automation/web?tab=runs&runId=123
/automation/web?tab=batches&batchId=456
```

## 公开分享链接

在运行报告或批次报告中点击“公开分享”，选择有效期后点击“生成公开链接”。

公开链接特点：

- 不需要登录。
- 只读访问，不能编辑、运行、删除用例。
- 支持 1 天、7 天、30 天、永久有效。
- 可以撤销。
- 可以重新生成，重新生成后旧 token 失效。
- 报告中的变量来自执行快照，敏感变量保持脱敏。

示例：

```text
/share/web-ui/report?token=webui_share_xxx
```

## 查看公开报告

打开公开链接后，可以看到：

- 运行报告：用例名称、结果、浏览器、耗时、步骤明细、失败证据、截图。
- 批次报告：批次名称、成功率、用例运行列表、失败摘要。

公开报告页面不会显示：

- 平台侧边栏。
- 工作空间切换。
- 编辑、运行、删除按钮。
- 用例配置的可写入口。

## 撤销公开链接

在报告的“公开分享”弹窗中找到启用中的链接，点击“撤销”。

撤销后：

- 已发出去的公开链接会立即失效。
- 再访问会返回不可访问/不存在。
- 非公开登录链接不受影响。

## 重新生成公开链接

平台只保存公开 token 的哈希值，不保存明文 token。

因此关闭弹窗后，如果忘记复制旧公开链接，不能再次查看旧 token。可以点击“重新生成”：

- 系统会生成一个新的公开链接。
- 旧 token 会失效。
- 新链接会自动复制到剪贴板。

## 安全说明

公开分享链接的安全边界：

- 后端只存 token 哈希，不存明文 token。
- 公开接口单独使用 `/api/public/automation/web/report-shares/**`。
- 公开接口只返回报告数据，不提供任何写操作。
- 截图下载必须携带有效 token。
- 过期或撤销的 token 不能访问报告和截图。
- 敏感变量按执行快照脱敏展示。

需要注意：

- 平台无法自动识别截图里的业务敏感信息。
- 如果被测页面截图包含客户信息、手机号、订单号等敏感内容，不建议生成公开链接。
- 永久有效链接应谨慎使用，优先选择 7 天有效期。

## 常见问题

### 公开链接打不开

可能原因：

- 链接已过期。
- 链接已被撤销。
- token 被复制不完整。
- 后端服务未启动。

### 截图打不开

可能原因：

- 报告没有生成截图。
- 截图文件不存在或被清理。
- 公开链接已过期或被撤销。

### 生成后忘记复制链接

由于平台不保存明文 token，旧链接不能再次查看。请在“公开分享”弹窗中点击“重新生成”。
