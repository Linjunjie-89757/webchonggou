# Web UI 用例复用使用说明

本文说明 Web UI 自动化用例的复制、模板新建、JSON 导入和 JSON 导出能力。

## 1. 复制为新用例

在 Web UI 用例列表中，点击目标用例右侧的“复制为新用例”。

系统会读取原用例详情，并打开用例编辑器：

- 用例 ID 会清空，保存时创建一条新用例。
- 用例名称会自动追加“副本”。
- 步骤会完整复制，但步骤 ID 会清空。
- 运行历史、最近结果、报告记录不会复制。

适用场景：

- 同一个流程在不同模块下稍作调整。
- 基于已有稳定用例快速扩展变体。
- 给同事复制一份草稿后再修改名称、模块、变量和断言。

## 2. 从模板新建

在 Web UI 页面顶部点击“从模板新建”，选择内置模板后，系统会打开用例编辑器。

当前内置模板：

- 登录流程
- 搜索/筛选/列表断言
- 表单新增
- 表格操作
- 文件上传

模板中的定位器和路径是示例值，需要在保存前改成目标系统真实页面的地址、定位器和变量名。

建议用法：

- 先选最接近业务流程的模板。
- 修改 `baseUrl`、路径、CSS/Text/Label 定位器。
- 将账号、密码、关键字等输入值改成 `{{变量名}}`。
- 调试运行通过后再保存成正式用例。

## 3. 导出 JSON

在用例列表中点击“导出 JSON”。

导出的 JSON 包含：

- 用例名称、模块、描述、起始地址。
- 浏览器类型、无头模式、默认超时、启停状态。
- 步骤类型、定位方式、定位器、输入值、截图策略、启用状态、排序。

导出的 JSON 不包含：

- 用例 ID。
- 步骤 ID。
- 最近运行结果。
- 执行报告。
- 截图证据。

这样可以避免导入时覆盖已有数据，也方便在不同环境之间传递用例草稿。

## 4. 导入 JSON

在 Web UI 页面顶部点击“导入 JSON”，粘贴导出的 JSON 内容，然后点击“校验并打开编辑器”。

导入规则：

- 只创建新用例，不覆盖已有用例。
- 校验通过后先打开编辑器，用户确认后再保存。
- 至少需要 1 个步骤。
- 至少需要 1 个启用步骤。
- CLICK、FILL、ASSERT_TEXT 等需要定位器的步骤必须有定位器类型和定位器。
- OPEN、FILL、ASSERT_TEXT、ASSERT_URL 等需要输入值的步骤必须有输入值或期望值。
- 不支持的浏览器类型会默认改为 Chromium。

如果校验失败，页面会直接列出失败原因，修正 JSON 后再导入即可。

## 5. 当前边界

本阶段暂不支持：

- 覆盖导入已有用例。
- 批量导入多个 JSON。
- Excel 导入/导出。
- 后端模板库管理。
- AI 生成步骤。

这些能力可以放到后续目标包继续扩展。当前版本优先保证“复制、模板、单用例 JSON 流转”足够稳定，方便同事快速创建和复用 Web UI 用例。
