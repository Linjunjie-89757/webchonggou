# Web UI 自动化第一阶段试用说明

这份说明给第一次试用 Web UI 自动化的同事使用。第一阶段目标是先把 Playwright 执行链路跑通：能创建用例、调试、正式运行、批量运行、看报告、下载截图、分享报告，并能继承配置中心里的 Web UI 环境和变量集。

## 1. 创建第一条 Web UI 用例

1. 进入 `自动化测试 -> Web UI 自动化 -> 用例列表`。
2. 点击 `新建用例`。
3. 填写用例名称、模块、起始地址、浏览器、默认超时。
4. 在步骤区点击 `新增步骤`，至少添加一个启用步骤。
5. 点击 `保存`。

建议第一条用例先从简单页面开始，例如：

| 顺序 | 步骤类型 | 建议内容 |
| --- | --- | --- |
| 1 | OPEN | `/login` 或完整登录页地址 |
| 2 | FILL | 输入用户名 |
| 3 | FILL | 输入密码 |
| 4 | CLICK | 点击登录按钮 |
| 5 | ASSERT_URL / ASSERT_TEXT | 验证登录成功 |

## 2. 常用步骤类型

| 类型 | 用途 | 必填项 |
| --- | --- | --- |
| OPEN | 打开页面 | 地址 |
| CLICK | 点击元素 | 定位方式、定位值 |
| FILL | 输入文本 | 定位方式、定位值、输入文本 |
| CLEAR | 清空输入框 | 定位方式、定位值 |
| HOVER | 鼠标悬停 | 定位方式、定位值 |
| PRESS_KEY | 键盘按键 | 按键值，例如 `Enter` |
| SELECT | 下拉选择 | 定位方式、定位值、选项值 |
| WAIT_FOR | 等待元素出现 | 定位方式、定位值 |
| ASSERT_VISIBLE | 断言元素可见 | 定位方式、定位值 |
| ASSERT_TEXT | 断言元素文本 | 定位方式、定位值、期望文本 |
| ASSERT_URL | 断言当前 URL | 期望 URL 片段 |
| ASSERT_TITLE | 断言页面标题 | 期望标题文本 |
| ASSERT_ATTRIBUTE | 断言属性 | 定位方式、定位值、`属性名=期望值` |
| ASSERT_COUNT | 断言元素数量 | 定位方式、定位值、`=1` / `>0` / `<3` |
| SCREENSHOT | 主动截图 | 无 |

## 3. 定位方式建议

| 定位方式 | 适合场景 | 示例 |
| --- | --- | --- |
| CSS | 最通用，推荐优先使用稳定 class、id、data 属性 | `#submit`、`[data-testid="login"]` |
| 文本 | 按钮或链接文案稳定时使用 | `登录` |
| 角色 | 更接近用户行为，适合按钮、链接、输入框 | `button:登录` |
| Label | 表单项有清楚 label 时使用 | `用户名` |
| XPath | 页面没有稳定属性时兜底 | `//button[text()='登录']` |

优先级建议：`data-testid / CSS` > `角色 / Label` > `文本` > `XPath`。XPath 能救急，但页面结构变化时更容易失效。

## 4. 环境和变量

Web UI 运行时可以选择运行环境。环境提供 Base URL、浏览器、无头模式、默认超时，也可以绑定默认变量集。

变量使用 `{{变量名}}` 引用，例如：

```text
用户名输入值：{{USERNAME}}
密码输入值：{{PASSWORD}}
```

变量优先级第一阶段为：

```text
内置变量 < 环境变量 < 变量集变量 < 运行时变量
```

运行报告会保存执行时刻的上下文快照，敏感变量会显示为 `******`。如果运行时手动选择变量集，会覆盖环境绑定的默认变量集。

## 5. 调试运行

编辑器里可以直接调试当前用例。调试前如果存在未保存修改，系统会提示本次将按当前草稿执行。

调试失败后会自动打开报告抽屉。重点看三块：

1. 失败摘要：快速判断失败原因。
2. 失败证据：步骤名、类型、定位器、输入值、错误信息。
3. 截图证据：有截图时可以打开或下载。

失败后可以点击 `定位步骤` 回到编辑器对应步骤继续调整。

## 6. 正式运行和批量运行

单条用例点击操作列里的 `运行`。运行完成后会自动打开执行报告。

批量运行先勾选多条用例，再点击 `批量运行`。批次报告会展示：

| 信息 | 说明 |
| --- | --- |
| 总数 / 成功 / 失败 | 本批次整体结果 |
| 失败用例 | 优先展示失败用例和失败摘要 |
| 运行记录 | 每条用例对应的 Run 报告 |
| 报告摘要 | 可复制给同事或贴到流水线日志 |

## 7. CI Token 和 Jenkins 接入

进入 `批次报告` 页签，在 `CI 触发入口` 创建 Token。明文 Token 只显示一次，请复制到 Jenkins 凭据或安全配置中。

接口地址：

```text
POST /api/automation/web/ci/batches/run
Authorization: Bearer <WEB_UI_CI_TOKEN>
Content-Type: application/json
```

请求体示例：

```json
{
  "workspaceCode": "risk-ops",
  "batchName": "Jenkins Web UI",
  "caseIds": [101, 102],
  "environmentId": 1,
  "variableSetId": 2,
  "headless": true,
  "stopOnFailure": false,
  "externalBuildId": "jenkins-${BUILD_NUMBER}"
}
```

接口返回里会包含 `reportUrl`，可从 Jenkins 日志直接跳回平台报告。

## 8. 报告分享

运行报告和批次报告都支持：

| 方式 | 用途 |
| --- | --- |
| 复制报告链接 | 需要登录平台后查看 |
| 公开分享 | 生成可匿名查看的只读链接 |

公开分享适合公司内部快速传阅。后续如果安全要求变化，可以关闭或收紧公开链接策略。

## 9. 常见失败排查

| 失败现象 | 排查建议 |
| --- | --- |
| 页面打开失败 | 检查 Base URL、OPEN 地址、网络、证书、登录前置条件 |
| 定位器未找到 | 用浏览器开发者工具确认 CSS/XPath，或改用 role/label/text |
| 等待超时 | 增大步骤超时，确认元素是否异步加载或被遮挡 |
| 断言失败 | 对照报告中的实际值和期望值，确认断言是否过严 |
| 变量缺失 | 检查环境默认变量集、运行时变量、变量名大小写 |
| 密码显示 `******` | 正常，敏感变量在报告里会脱敏 |
| CI 触发 401 | 检查 Bearer Token 是否复制完整、是否已禁用或重新生成 |

## 10. 第一阶段边界

第一阶段以 Playwright 为执行引擎。Selenium、Cypress、多引擎切换、录制生成步骤、AI 生成步骤属于后续阶段，不在本轮试用范围内。
