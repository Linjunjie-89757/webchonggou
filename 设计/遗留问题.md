# 遗留问题

## 场景步骤保存模型

- 当前处理：步骤抽屉点击保存时，会保存整个场景，保证用户修改步骤后直接运行能使用最新配置。
- 后续优化：后端可以新增单步骤保存接口，例如 `PATCH /api/automation/api/scenarios/{scenarioId}/steps/{stepId}`，只替换 `stepsJson` 中被编辑的步骤。
- 需要注意：单步骤保存要处理嵌套步骤、步骤排序、并发覆盖、场景版本冲突和保存失败提示。

## Path 参数执行闭环

- 现状：Swagger/OpenAPI 导入的路径参数会保留为 `/detail/{id}`，运行时只替换 `{{变量}}` / `${变量}`，不会自动替换 `{id}`。
- 风险：如果用户没有手动把 `{id}` 改成真实值，执行时会在 `URI.create()` 报非法路径。
- 后续优化：导入时把 `in: path` 参数转成可填写的路径参数；复制到场景/用例时保留；执行前用填写值替换 `{id}`，未填写时提示“路径参数 id 必填”。
