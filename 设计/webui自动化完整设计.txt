平台 Web UI 自动化 —— 完整设计
一、整体定位
在自动化测试平台内，与现有 API 自动化并列，作为独立的自动化能力。

底层执行引擎使用 Playwright，浏览器默认 Chromium，同时支持 Firefox、WebKit。

平台负责用例管理、调度、报告、权限，真正跑测试的是可横向扩展的执行节点。

与 API 自动化深度联动：Web UI 用例中可以直接插入 API 步骤，用于快速准备数据或清理环境。

二、核心模块设计
1. 用例管理模块
1.1 用例属性

字段	说明
用例名称	必填，唯一建议加项目内唯一约束
模块/二级模块	树形结构，支持多级分类
标签	多标签，便于按业务线/场景筛选和批量执行
描述	富文本，说明用例目的
Base URL	用例默认被测地址
浏览器	下拉选择
无头模式	是/否
默认超时	单位秒
是否启用	禁用后无法执行
公开前置用例	可引用1个公共前置用例
公开后置用例	可引用1个公共后置用例
失败重试次数	0-3次，非断言失败时可自动重试
截图策略	不截图/仅失败截图/每步截图
失败处理策略	遇到失败：停止用例 / 继续执行后续步骤
版本号	自动递增，记录修改人与时间
1.2 配置优先级（强制明确，界面标注）

text
运行参数(手动选择) > 环境配置 > 用例默认配置
用例页面上，会被环境覆盖的字段旁显示 “可被环境覆盖” 标签。

运行时如果选择了环境，会自动用环境对应字段覆盖用例的默认值（除非运行参数又覆盖了一次）。

1.3 公共用例

公共前置/后置用例本质上就是普通用例，被引用时自动嵌入步骤流的前/后。

修改公共用例的步骤，所有引用它的用例自动生效。

支持导入/导出（JSON/Excel），方便跨项目迁移。

1.4 用例列表

支持按模块、标签、名称、创建人筛选

批量操作：启用、禁用、删除、批量执行

2. 步骤编辑器
2.1 步骤类型（完整清单）

基础交互

OPEN 打开页面 (URL 必填)

CLICK 点击 (定位器必填)

FILL 输入 (定位器 + 值必填)

CLEAR 清空输入框 (定位器)

HOVER 悬浮

DOUBLE_CLICK 双击

RIGHT_CLICK 右键

DRAG 拖拽 (源定位器 + 目标定位器)

PRESS_KEY 按键 (键盘按键，支持组合键如 Control+A)

SELECT 下拉选择 (定位器 + 选项值/索引)

特殊场景

SWITCH_IFRAME 切换到 iframe (定位器)

SWITCH_WINDOW 切换到指定窗口 (索引/title)

NEW_TAB 打开新标签页 (URL)

CLOSE_TAB 关闭当前标签页

FILE_UPLOAD 文件上传 (定位器 + 文件路径)

HANDLE_ALERT 处理弹窗 (accept/dismiss/输入文本)

等待

WAIT_FOR 等待元素可见 (定位器 + 超时)

WAIT_FOR_URL 等待 URL 包含指定字符串

WAIT_FOR_TITLE 等待页面标题包含指定文本

WAIT_FOR_NETWORK 等待网络空闲

WAIT_FOR_LOAD_STATE 等待页面加载状态 (load/domcontentloaded)

SLEEP 固定等待 (秒)

断言

ASSERT_VISIBLE 元素可见

ASSERT_HIDDEN 元素不可见

ASSERT_TEXT 元素文本等于/包含

ASSERT_URL 当前 URL 包含/等于

ASSERT_TITLE 页面标题包含/等于

ASSERT_ATTRIBUTE 元素属性等于/包含 (如 disabled, checked)

ASSERT_COUNT 定位器匹配元素数量等于/大于/小于

数据与变量

EXTRACT 提取元素文本/属性/值 -> 存入变量

SET_VARIABLE 直接设置变量

API_REQUEST 发送 HTTP 请求（与 API 自动化复用），返回结果可存入变量

2.2 变量系统

变量引用格式：${变量名}

变量来源：

环境变量（环境配置中维护，如账号、密码）

全局变量（跨用例，可在运行时传入）

用例变量（步骤内提取或设置，步骤间传递）

内置变量（如 ${TIMESTAMP}, ${RANDOM_STRING}）

数据驱动：同一条用例可绑定多组数据（JSON/CSV），每组数据作为全局变量注入，自动循环执行，直到数据用完或失败终止。

2.3 逻辑控制

条件步骤（IF）：如果定位器存在/变量等于某值，则执行子步骤块。

循环步骤（FOR）：遍历数据列表，或对定位器匹配的多个元素逐个操作。

失败策略：单步失败时，可选择「停止执行」「跳过继续」「重试N次后停止」。

2.4 定位器链式组合

一个步骤可配置：父定位器 + 子定位器。

对应 Playwright 的 parentLocator.locator(child)，解决列表行内操作按钮的问题。

2.5 元素库引用

步骤可直接从“元素库”选择元素，无需重复编写定位器。

页面改版只需更新元素库，所有引用步骤自动生效。

2.6 编辑器辅助功能

定位器验证：点击按钮后，后台启动浏览器打开对应 Base URL+当前步骤页面，返回匹配数量及元素高亮截图。不用跑整条用例。

定位器生成（进阶，可后期做）：在调试预览页面直接点击元素，自动回填最佳定位器。

步骤操作：复制、上移、下移、禁用/启用、折叠展开。

步骤注释：每个步骤可添加注释，便于协作。

3. 定位方式库
支持优先级排序（界面提供引导）：

Test ID (data-testid) —— 最稳定

Role (ARIA role)

Label (关联标签)

Placeholder

CSS (选择器)

XPath

Text (纯文本匹配)

编辑器内提供下拉选择定位方式，并提示“建议优先使用 Test ID 或 Role”。

4. 环境配置
4.1 环境基础字段

环境名称

Base URL

浏览器类型

无头模式

默认超时

启用/停用

4.2 环境变量

以键值对形式维护，如 USERNAME=admin, PASSWORD=123456。

环境变量可在用例步骤中通过 ${USERNAME} 引用。

不同环境切换时，变量值自动切换，用例无需改动。

4.3 高级浏览器配置

视口大小（宽x高），支持预设手机/平板分辨率

忽略 HTTPS 错误

语言/时区设置

录制视频（可选：仅失败/全程）

5. 运行与调试
5.1 单用例运行

列表页可直接点击“运行”，弹出快速配置：选择环境、无头模式、变量覆盖。

运行后可立即打开报告。

5.2 调试模式

编辑器内点击“调试运行”，默认强制非无头、慢速执行。

步进执行：跑一步停一步，展示当前步骤截图、变量值，可手动点击“下一步”或“停止”。

调试过程中可暂停并触发“元素拾取”辅助写定位器。

完成后自动打开调试报告（包含 Trace 和截图）。

5.3 排障产物

失败/调试时强制生成 Playwright Trace（trace.zip），可在线回放完整时间线。

支持开启失败录屏或全程录屏。

报告页可查看控制台错误、网络请求失败详情。

6. 执行报告
6.1 单用例报告

基本信息：执行结果、浏览器、环境、Base URL、耗时

步骤明细：序号、步骤类型、定位器、输入值、期望值、实际结果、状态、截图、错误信息

失败自动分类：元素未找到、断言失败、超时、网络错误、JS错误

附加信息：控制台日志、失败请求瀑布流、Trace在线查看器、录屏播放器

6.2 批次报告

来源：手动 / CI / 定时任务

统计：总数、成功、失败、跳过、通过率

环境、执行时间、外部构建号

用例列表：每个用例的状态、耗时、失败摘要，可下钻到单用例报告

失败用例 Top 排行

支持导出 PDF/HTML，支持一键分享链接

6.3 通知

批量执行完成后，可选择推送到企业微信、钉钉、邮件（可配置通知条件：仅失败时通知或始终通知）。

7. 批量执行与调度
7.1 批量运行

用例列表多选 -> 批量运行，需配置批次名称、环境、无头模式、并发数、失败即停开关。

支持按标签、模块快速全选。

7.2 并发控制

配置最大并发浏览器实例数（默认2，可按执行节点资源调整）。

并行执行时，每个用例使用独立浏览器上下文，互不干扰。

超过并发数的用例排队等待。

7.3 定时任务

平台内可创建定时任务，指定 Cron 表达式、用例集、环境等，直接由调度器触发，不依赖 CI。

定时任务同样生成批次报告，可配置通知。

8. CI 集成
8.1 触发接口

URL: POST /api/v1/webui/run-batch

鉴权：Bearer Token（Header: Authorization: Bearer <token>）

请求参数（JSON）：

json
{
  "batch_name": "CI Build #123",
  "environment_id": "env-001",
  "headless": true,
  "concurrency": 4,
  "stop_on_failure": true,
  "case_tags": ["smoke", "login"],
  "case_ids": [],
  "variables": {"ENV": "staging"},
  "callback_url": "https://jenkins.company.com/callback"
}
返回：批次 ID，可查询状态。

8.2 Token 管理

仅登录用户可管理 Token（创建、禁用、删除、重新生成）。

创建时明文仅展示一次，数据库只存储 SHA256 哈希。

可配置 Token 作用范围：允许操作的项目、允许使用的环境。

支持设置 Token 有效期。

8.3 回调 Webhook

如果传了 callback_url，批次执行终态（成功/失败/部分失败）时，平台向该 URL 发送 JSON 结果，含批次 ID、通过率、报告链接、每个用例状态。

支持自定义 Header 和重试策略。

9. 元素库（Page Object 管理）
按页面组织，例如“登录页”、“订单列表页”。

每个元素包含：元素名称、定位方式、定位器值、描述。

支持版本管理，修改后历史可追溯。

用例步骤中通过“选择元素”引用，切换环境时元素定位器无需改动（因为与环境无关）。

提供“扫描页面”功能（扩展），自动抓取页面上带 data-testid 或 role 的元素入库。

10. 项目与权限
10.1 多项目隔离

所有对象（用例、环境、元素库、Token、报告）均归属到项目。

用户登录后只能看到有权限的项目数据。

10.2 角色权限

管理员：管理项目成员、Token、所有删除操作

测试开发：创建/编辑用例、环境、元素库，运行调试

测试人员：运行用例（需指定环境），查看报告

只读：查看用例、报告，不能修改和执行

三、等待与稳定性策略（全局）
全局默认等待策略：state: 'domcontentloaded' + 自动等待元素可操作状态（Playwright 原生自动等待），除非步骤显式要求更严格条件。

步骤中超时继承：环境默认超时 > 用例默认超时 > 步骤单独超时（可配）。

网络不稳定重试：用例级别可配置失败重试 0-3 次，仅当失败原因为 TimeoutError 或 NetworkError 时触发。

四、执行架构（预留分布式）
平台服务：负责 API、Web 界面、任务调度、结果存储。

执行节点 Worker：独立进程，只运行 Playwright，通过消息队列（如 Redis/直接 HTTP）领取任务，上报状态和产物。

Worker 可部署在多台机器，横向扩容。

MVP 阶段可将 Worker 与平台放同一容器，但接口设计上隔离，便于后续拆分。

五、落地路线图（满分版推荐顺序）
Phase 1 — 核心闭环（必需）

用例管理 + 步骤编辑器（基础8步+ifreme/window/select/hover/alert）

定位器验证 + 变量系统 + 公共前置/后置

环境配置（含环境变量）+ 配置优先级

单用例运行 + 调试步进 + Trace/截图

单用例报告 + 批次报告

批量运行 + CI Token 触发（最小集）

Phase 2 — 稳定性与效率（上线后1-2迭代）

元素库

数据驱动

并发执行

定时任务

通知推送

API 步骤联动

失败重试、高级等待策略

Phase 3 — 规模化与体验（长期）

分布式执行节点

实时元素拾取、录屏

统计大盘、趋势分析

精细权限体系与审计日志

导入导出、跨项目共享元素库









以下是基于「满分设计」方案，为你梳理的 Web UI 自动化平台前端设计。包含整体布局、各模块页面设计、关键组件、交互流程和界面结构图（文字描述 + 可用 ASCII 辅助理解）。

一、整体布局框架
采用经典的企业级后台布局：顶部导航 + 侧边菜单 + 主内容区。

text

| 侧边栏  |                                                         |
| 菜单项  |            主内容区 (动态渲染对应模块页面)                |
|        |                                                         |

| -环境配置|                                                         |
| -元素库  |                                                         |
| -执行报告|                                                         |
| -批次报告|                                                         |
| -CI集成  |                                                         |

| -系统设置|                                                         |
+--------+---------------------------------------------------------+
顶部栏功能：

左侧 Logo + 平台名称。

项目切换下拉框：当前所在项目，切换后全局数据刷新。

全局搜索：可搜索用例名称/ID、报告ID。

通知图标：批次执行完成、失败告警等。

用户头像下拉：个人设置、API Token 快捷入口、退出。

侧边菜单：

采用树形或分组展开，当前模块高亮。

支持折叠，给主内容区更多空间。

二、各模块页面详细设计
1. 用例管理模块
1.1 用例列表页
布局：上方条件筛选栏 + 操作按钮栏 + 下方表格。

筛选栏：

搜索框（用例名称模糊搜索）

模块树形下拉选择（一级/二级模块）

标签多选下拉

状态：启用/禁用/全部

创建人筛选

操作按钮：

[新建用例] 主按钮

[批量操作]：批量启用、批量禁用、批量删除、批量运行（需选中用例）

[导入] / [导出] 按钮（导出为 JSON/Excel）

表格列：

复选框（用于批量操作）

用例名称（可点击跳转详情/编辑器）

所属模块

标签（展示为彩色标签组件）

前置/后置用例（有则显示链接）

最近修改人 + 时间

状态（启用/禁用开关，可直接切换）

操作列：编辑、复制、删除（二次确认）、运行（弹出快速运行配置）

交互细节：

运行按钮：弹出对话框选择环境、无头模式、变量覆盖等，点击“开始运行”，页面上方出现 toast 通知，并在“执行报告”中生成记录。

批量运行：选中多行后点击“批量运行”，同样弹出配置对话框（与批量运行页一致）。

列表支持分页，可调整每页条数。

1.2 用例详情/编辑器页
布局：左侧步骤列表面板 + 右侧步骤编辑区域 + 上方用例基本信息头。

页面顶栏（用例基本信息卡片，可折叠）：

用例名称（可编辑）、模块、标签、描述。

默认配置组：Base URL、浏览器、无头模式、超时等，每个字段旁带 “可被环境覆盖” 提示图标。

前置/后置用例选择器。

失败重试次数、截图策略、失败处理策略下拉。

[保存基本信息] [运行] [调试运行] [查看历史版本] 按钮。

左侧步骤列表面板：

步骤列表按序号垂直排列，可拖拽排序。

每个步骤项显示：序号、类型图标、简述（如“CLICK 登录按钮”）、状态（启用/禁用开关）。

选中某步骤时右侧显示其编辑表单。

底部有 [添加步骤] 按钮，点击弹出步骤类型选择面板。

支持步骤复制、上移、下移、删除、禁用/启用。

支持定位器验证按钮（仅对需要定位器的步骤出现，点击后右侧预览区显示高亮结果）。

右侧步骤编辑区域：

根据步骤类型动态渲染表单：

通用：步骤注释输入框。

定位器选择：定位方式下拉 + 定位器值输入框；支持切换到「链式组合」，出现父定位器输入行。

值输入：对于FILL、ASSERT_TEXT、SELECT等，显示输入值/期望值，支持变量引用 ${}。

特殊字段：文件路径选择（FILE_UPLOAD）、按键下拉（PRESS_KEY）等。

编辑区域下方有“验证定位器”按钮（动态出现），点击后调用后端，在右侧或弹出层显示结果（匹配数量 + 高亮截图）。

变量管理小侧边：可打开变量面板，展示当前用例已有变量，方便复制变量名。

调试模式相关：

点击“调试运行”后，编辑器进入调试模式：步骤列表旁边出现状态指示，当前执行步骤高亮，右侧展示该步的实时截图、变量值。提供“下一步”、“停止”按钮。

交互：

步骤列表可通过上下按钮调整顺序，或拖拽。

每当步骤类型改变，表单自动清除不必要的字段。

添加步骤时，弹出类型选择浮层，包含分类（基础交互、特殊场景、等待、断言、数据变量），支持搜索。

定位器验证结果用弹出层展示：顶部显示“匹配到 N 个元素”，下方是高亮截图，用户可以关闭或重新验证。

2. 环境配置模块
2.1 环境列表页
表格：环境名称、Base URL、浏览器、无头模式、超时、启用状态、操作（编辑/复制/删除）。

[新建环境] 按钮。

四、关键组件设计思路
1. 步骤类型选择器
弹出层，支持分类标签（如：交互、等待、断言、数据），每类下列举类型卡片，卡片上有图标和简短描述。

值字段可切换明文/密文显示（对密码类变量掩码）。

6. 定时任务 Cron 输入器
提供常用预设（每分钟、每小时、每天凌晨、每周一），并支持手动输入表达式。

实时预览人类语言描述和接下来5次执行时间。

六、前端交互原则补充
即时反馈：任何操作（如保存、运行、验证）都应通过 loading、toast、进度条给予用户反馈，避免长时间无响应。

容错性：删除操作必须二次确认；运行中禁用编辑和运行按钮；切换模块前提示未保存更改。

可配置性：表格列显示可定制；用户可保存常用筛选条件。

响应式：主内容区弹性布局，编辑器左右面板可拖拽调整宽度。

键盘快捷键：编辑器支持 Ctrl+S 保存；调试模式空格键下一步等（进阶）。

国际化预留：前端所有文案使用 i18n 方案，便于未来多语言（但不是MVP要求）。

七、技术栈推荐（前端）
框架：Vue 3 + TypeScript 或 React + TypeScript，搭配状态管理（Pinia / Redux Toolkit）。

UI 组件库：Element Plus (Vue) 或 Ant Design (React)，具备表格、树、表单、弹窗、标签、步骤条等开箱即用组件。

代码编辑器：Monaco Editor 可用于 JSON/变量输入。

Trace Viewer：若要在 Web 上直接查看 Playwright Trace，需集成 Playwright 的 trace-viewer 包（通过 iframe 或组件加载）。

拖拽：vuedraggable 或 react-beautiful-dnd 用于步骤排序和元素库页面排序。

图表：ECharts 用于批次报告趋势统计（后期）。

仅登录用户可管理，创建后明文仅展示一次，数据库存 SHA256

可配置作用范围：允许的项目、环境

支持有效期设置

回调 Webhook

若传入 callback_url，批次终态时向该 URL POST 结果（含通过率、报告链接、各用例状态）

2.9 元素库（Page Object 管理）
按页面组织，元素包含：名称、定位方式、值、描述

支持版本管理，修改可追溯

用例通过“选择元素”引用，页面改版只改元素库

支持“扫描页面”功能自动抓取 Test ID 或 Role

2.10 项目与权限
多项目隔离

用例、环境、元素库、Token、报告均归属项目

角色

管理员：管理成员、Token、删除操作

测试开发：创建/编辑用例、环境、元素库，运行调试

测试人员：运行用例，查看报告

只读：查看，不能修改和执行

三、全局稳定性策略
默认等待策略：domcontentloaded + Playwright 自动等待元素可操作

超时继承：环境默认 > 用例默认 > 步骤单独超时

失败重试：仅当失败原因为 TimeoutError 或 NetworkError 时触发

四、执行架构（预留分布式）
平台服务：API、界面、任务调度、结果存储

执行节点 Worker：独立进程，运行 Playwright，领取任务，上报状态和产物

MVP 阶段可同容器部署，但接口隔离，便于后续拆分
