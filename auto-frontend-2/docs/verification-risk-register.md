# Verification And Risk Register

This file tracks known unverified paths and residual risks during the frontend rebuild. Update it after each goal when a path is intentionally left for later regression.

## Current Policy

- Each feature goal runs lightweight validation: `npm.cmd run typecheck`, `npm.cmd run build`, and focused browser smoke checks when useful.
- Full Playwright regression is reserved for module stabilization goals.
- Avoid real create/update/delete operations on production-like data unless test data or explicit approval is available.
- Backend compile is only required when backend files are changed.

## Recommended Next Regression Goals

1. Defect management stabilization: full smoke plus real-data issue list for read, filter, create/edit, detail, assign, transition.
2. Defect UI parity: compare density, operation menu, dialogs, drawer, empty/error states against old frontend.
3. Cross-module session regression: login, 401, logout, protected route redirect, and modal save failure behavior.
